## list.md
### album file
https://github.com/bpmbpm/family-tree/pull/123  
При запуске из desktop (file:// protocol) галерея альбомов показывает «Файлы не найдены в папке album/», хотя файл `album/list.md` присутствует в архиве.

В PR #121 был добавлен асинхронный XMLHttpRequest как запасной вариант, но проблема всё равно воспроизводится. Это происходит потому, что в некоторых браузерах (Chrome при запуске с file://) **асинхронный XHR тоже не работает** для локальных файлов — срабатывает `onerror` или `onload` с пустым `responseText`.

## Причина

Функция `showAlbumGallery()` использовала два способа загрузки:
1. `fetch('album/list.md')` — блокируется браузером на file://
2. Асинхронный `XMLHttpRequest` — также может не сработать в Chrome на file://

Функция `loadConfig()` в том же файле использует три способа, включая **синхронный XHR** (`xhr.open('GET', ..., false)`), который надёжно работает для локальных файлов в большинстве браузеров.

## Решение

По аналогии с `loadConfig()` добавлен третий запасной вариант — синхронный XHR. Теперь последовательность:
1. `fetch()` — для HTTP-серверов
2. Асинхронный XHR — запасной вариант для file://
3. **Синхронный XHR** — третий запасной вариант, если асинхронный не сработал
   
#### test
Открыть experiments/test_sync_xhr_album.html через file:// — файл проверяет все три способа загрузки и показывает, какой из них сработал.
