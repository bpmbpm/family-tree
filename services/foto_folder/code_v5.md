Сделай два раздельных файла .html - один для https://github.com/ второй для desktop.  
В прошлой реализации при локальном запуске все равно файлы находились в папках, но выдавало как будто - не найдена. Для версии на https://github.com кнопка Проверить лишние файлы в папке ... - лишняя

Мы должны создать два отдельных HTML-файла: один для работы на GitHub Pages (онлайн), другой для локального использования (desktop). В локальной версии нужно корректно работать с файловой системой (проверять наличие файлов, искать лишние). В онлайн-версии проверка лишних файлов невозможна из-за ограничений браузера, поэтому эта функция должна отсутствовать или быть заменена информационным сообщением. Также в локальной версии необходимо использовать механизмы, позволяющие проверять наличие файлов (например, через Image для file://).

Предыдущая реализация показала, что в локальном режиме с file:// проверка через fetch не работает, но через Image работает. Также для проверки лишних файлов нужно использовать <input type="file" webkitdirectory>, что позволяет получить список файлов в выбранной папке (работает только в некоторых браузерах, но это допустимо для desktop).

Таким образом, сделаем два файла:

service_foto_github.html – для GitHub Pages. Без кнопок проверки лишних файлов, только проверка наличия по idA (с возможностью ручной загрузки Excel, если не сработает автоматическая). Вывод информации, что лишние файлы проверить нельзя.

service_foto_desktop.html – для локального использования. Включает кнопки для проверки лишних файлов, использует fileExists через Image, и предоставляет инструкции по выбору папок.

Оба файла будут иметь общую структуру, но с различиями в функциональности.
