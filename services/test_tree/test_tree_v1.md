# Проверка файла tree.xlsx (test_tree_v1)

Документация по инструменту проверки целостности данных семейного древа.

## Расположение файлов

- **HTML-инструмент**: `ver3/test_tree_v1.html`
- **Документация**: `services/test_tree/test_tree_v1.md`

## Использование

1. Откройте файл `test_tree_v1.html` в браузере
2. Нажмите "Загрузить tree.xlsx" для автоматической загрузки файла из той же директории
3. Или нажмите "Выбрать файл..." для выбора файла вручную
4. Результаты проверки отобразятся на странице с подробным отчётом

## Список проверок

### 1. Наличие обязательных листов
- **Описание**: Проверка наличия листов `person` и `family`
- **Тип**: Критическая
- **Условие успеха**: Оба листа присутствуют в книге

### 2. Заполненность idA в листе person
- **Описание**: Все персоны должны иметь заполненный идентификатор idA
- **Тип**: Критическая
- **Условие успеха**: Все строки имеют непустое значение idA (не формулу)

### 3. Заполненность idA в листе family
- **Описание**: Все семьи должны иметь заполненный идентификатор idA
- **Тип**: Критическая
- **Условие успеха**: Все строки имеют непустое значение idA (или формулу, вычисляющую husband-wife)

### 4. Ссылки husband и wife на person.idA
- **Описание**: Значения полей husband и wife в листе family должны существовать в листе person
- **Тип**: Критическая
- **Условие успеха**: Все значения husband и wife найдены в person.idA

### 5. Ссылки hasFather и hasMother на person.idA
- **Описание**: Родительские ссылки должны соответствовать существующим персонам
- **Тип**: Предупреждение (родители могут быть не внесены в базу)
- **Условие успеха**: Значения hasFather и hasMother найдены в person.idA или пусты

### 6. Уникальность idA в листе person
- **Описание**: Идентификаторы персон не должны повторяться
- **Тип**: Критическая
- **Условие успеха**: Все значения idA уникальны

### 7. Уникальность idA в листе family
- **Описание**: Идентификаторы семей не должны повторяться
- **Тип**: Критическая
- **Условие успеха**: Все значения idA уникальны

### 8. Корректность значений sex
- **Описание**: Поле sex должно содержать М (мужской) или Ж (женский)
- **Тип**: Критическая для некорректных значений, предупреждение для пустых
- **Условие успеха**: Все значения sex равны "М" или "Ж"

### 9. Отсутствие циклических ссылок
- **Описание**: Родительские связи не должны образовывать циклов
- **Тип**: Критическая
- **Условие успеха**: Нет циклических зависимостей через hasFather/hasMother

### 10. Корректность годов рождения и смерти
- **Описание**: Годы должны быть в разумных пределах, год смерти не раньше года рождения
- **Тип**: Критическая для death < birth, предупреждение для нереалистичных годов
- **Условие успеха**: Годы в пределах 1000-текущий год, death >= birth

### 11-12. Ссылки в листе foto_person
- **id_person** должен существовать в person.idA
- **id_personAll** (множественные значения через ";") все должны существовать в person.idA

### 13-14. Ссылки в листе foto_family
- **id_family** должен существовать в family.idA
- **id_personAll** (множественные значения через ";") все должны существовать в person.idA

### 15-16. Ссылки в листе foto_group
- **id_person** должен существовать в person.idA
- **id_personAll** (множественные значения через ";") все должны существовать в person.idA

### 17-18. Ссылки в листе foto_location
- **id_personAll** (множественные значения через ";") все должны существовать в person.idA
- **id_familyAll** (множественные значения через ";") все должны существовать в family.idA

### 19-20. Ссылки в листе event
- **id_personAll** (множественные значения через ";") все должны существовать в person.idA
- **id_familyAll** (множественные значения через ";") все должны существовать в family.idA

### 21. Ссылки на фото в event
- **Описание**: Поля foto_person, foto_family, foto_group, foto_location должны ссылаться на существующие записи в соответствующих листах
- **Тип**: Предупреждение
- **Условие успеха**: Все ссылки на фото найдены в соответствующих листах

## Формат множественных идентификаторов

Поля `id_personAll` и `id_familyAll` могут содержать несколько идентификаторов, разделённых символом ";" (точка с запятой). Пробелы вокруг разделителя игнорируются.

**Примеры:**
```
Ульянов_Владимир_Ильич
Ульянов_Владимир_Ильич; Свердлов_Яков_Михайлович
Ульянов_Владимир_Ильич ; Ульянов_Илья_Николаевич
```

## Альтернативная реализация проверок средствами Excel (без VBA)

Все проверки можно реализовать с помощью формул Excel:

### 1. Проверка заполненности idA в person

В новом столбце (например, T) листа person добавьте формулу:
```excel
=ЕСЛИ(ИЛИ(A2="";ЛЕВСИМВ(A2;1)="=");"Ошибка: idA пустой или формула";"OK")
```

### 2. Проверка заполненности idA в family

В новом столбце листа family:
```excel
=ЕСЛИ(ИЛИ(A2="";ЛЕВСИМВ(A2;1)="=");"Ошибка: idA пустой или формула";"OK")
```

### 3. Проверка husband в person.idA

В листе family добавьте столбец с формулой:
```excel
=ЕСЛИ(B2="";"";ЕСЛИ(СЧЁТЕСЛИ(person!$A:$A;B2)>0;"OK";"Ошибка: husband не найден"))
```

### 4. Проверка wife в person.idA

```excel
=ЕСЛИ(C2="";"";ЕСЛИ(СЧЁТЕСЛИ(person!$A:$A;C2)>0;"OK";"Ошибка: wife не найдена"))
```

### 5. Проверка hasFather в person.idA

В листе person:
```excel
=ЕСЛИ(E2="";"";ЕСЛИ(СЧЁТЕСЛИ($A:$A;E2)>0;"OK";"Предупреждение: отец не найден"))
```

### 6. Проверка hasMother в person.idA

```excel
=ЕСЛИ(F2="";"";ЕСЛИ(СЧЁТЕСЛИ($A:$A;F2)>0;"OK";"Предупреждение: мать не найдена"))
```

### 7. Проверка уникальности idA в person

```excel
=ЕСЛИ(СЧЁТЕСЛИ($A$2:$A$100;A2)>1;"Ошибка: дубликат";"OK")
```

### 8. Проверка значения sex

```excel
=ЕСЛИ(ИЛИ(C2="М";C2="Ж");"OK";ЕСЛИ(C2="";"Предупреждение: пол не указан";"Ошибка: некорректный пол"))
```

### 9. Проверка year death >= birth

```excel
=ЕСЛИ(И(G2<>"";I2<>"");ЕСЛИ(I2>=G2;"OK";"Ошибка: год смерти раньше рождения");"")
```

### 10. Проверка множественных ID (id_personAll)

Для проверки множественных ID требуется вспомогательная формула или макрос, так как стандартные функции Excel не работают с разделёнными значениями напрямую.

**Упрощённый вариант** (для первого ID в списке):
```excel
=ЕСЛИ(N2="";"";ЕСЛИ(СЧЁТЕСЛИ(person!$A:$A;СЖПРОБЕЛЫ(ЛЕВСИМВ(N2;ЕСЛИ(ЕОШИБКА(НАЙТИ(";";N2));ДЛСТР(N2);НАЙТИ(";";N2)-1))))>0;"OK";"Ошибка"))
```

### Использование условного форматирования

Для визуального выделения ошибок:

1. Выделите диапазон с формулами проверки
2. Главная → Условное форматирование → Правила выделения ячеек → Текст содержит
3. Введите "Ошибка" и выберите красную заливку
4. Повторите для "Предупреждение" с жёлтой заливкой

### Сводная проверка на отдельном листе

Создайте лист "Проверка" с итоговыми формулами:

```excel
| A | B |
|---|---|
| Проверка | Результат |
| Пустые idA в person | =СЧЁТЕСЛИМН(person!A:A;"")&" из "&ЧСТРОК(person!A:A)-1 |
| Дубликаты в person | =СУММПРОИЗВ((СЧЁТЕСЛИ(person!$A$2:$A$100;person!$A$2:$A$100)>1)/СЧЁТЕСЛИ(person!$A$2:$A$100;person!$A$2:$A$100)) |
| Отсутствующие husband | =СУММПРОИЗВ(--(family!B2:B100<>"")*(СЧЁТЕСЛИ(person!$A:$A;family!B2:B100)=0)) |
| Отсутствующие wife | =СУММПРОИЗВ(--(family!C2:C100<>"")*(СЧЁТЕСЛИ(person!$A:$A;family!C2:C100)=0)) |
```

## Технические детали

### Поддерживаемые браузеры
- Chrome 60+
- Firefox 60+
- Safari 12+
- Edge 79+

### Зависимости
- [SheetJS (xlsx)](https://sheetjs.com/) — библиотека для чтения Excel файлов

### Ограничения
- Формулы в ячейках idA отображаются как "=...", а не как вычисленные значения (ограничение SheetJS)
- Для больших файлов (>10000 строк) может потребоваться время на обработку

### Локальный запуск (file://)
При открытии через `file://` автоматическая загрузка tree.xlsx может не работать из-за ограничений безопасности браузера. Используйте кнопку "Выбрать файл..." или запустите локальный веб-сервер.
