### base
В файле excel tree.xlsx – веду три типа полей:
- имя поля используется кодом (js). С ними нужно быть особенно аккуратно (не менять названия полей, т.к. это потребует изменение кода). Их совсем немного
- имя поля не используется кодом (js) и не отображается в карточках (окнах UI, реализованных JS)
- имя поля не используется кодом и отображается в карточках (в таких полях на конце имени поля ставлю `_`)

Есть четвертый тип, у которого на конце A, например, `idA` – auto в ячейке формула и значение ячейки вычисляется на основе других.  
Получается алгоритм: хотим в excel (tree.xlsx) добавить новый параметр персоны, семьи, события или чего угодно (какого0либо другого листа), то просто дабавляем поле и вводим данные.  
Если нужно, чтобы это поле программа выводило в окно (UI), например, Основные свойствf person или family, то на конце имени поля добавляем `_` и эти значения и имя параметра будут выводиться (не забыть название поля указать на листе language)  
Когда захотим убрать отображение из карточки персоны или семьи, то просто убираем из названия поля последний `_`.   

Если конечно это поле нужно будет анализировать, то в программе нужно предусмотреть обработку, например, при обработке такого поля при наличии символа `_` на конце удалять его (чтобы был единый подход к обработке). 

### person window
Алгоритм работы окна, например, Основные свойствf person.  
Нужно показать карточку персоны, т.е. открыть "Основные свойствf person". В панеле окна отображается id (idA).

Последовательность отображения полей в окон - по очереди на листе (пока не сделано).

### add columns
При добавлении полей на листы excel для отображаемых полей добавлять их на лист language (RU + EN)  
Пример issue:

Окно "Основные свойства person" отражается верно: 
- для значения RU (config.js) подставляются в карточку правильная замена полям листа excel (см. лист language)
- поле hyperLink (hyperLink_) отображает только доменное имя второго уровня и с поддержкой множественного перечисления (разделитель ;)

Также напоминаю, что поля (название полей) в excel могут быть без `_` (нижнее подчеркивание) и c ним. Нижнее подчеркивание отвечает лишь за вывод поля в окно интерфейса. Обработка же логики должна быть неизменной, например, добавление `_` к названию поля должно повлиять только на отображение в окне (UI), но не на логику обработки (т.е. при обработке отбрасывай `_`, но анализируй только при выводе в окно).

Проверь отображение наименований для RU в других окнах, например, при открытии фотографий в окне отображаются title, label вместо названий, определяемых на листе language: 
foto_person - title_ - название
Пример некорректного отображения пояснений в окне просмотра фото: Лебедев_Сергей_Михайлович-учетно_послужная_карточка.jpg (foto_item).
Это видимо произошло из отсутствия на листе language в поле object данных для foto_item (также добавь для foto_event).

Кроме того, поле hyperLink (hyperLink_) отображено без сокращения до доменного имени второго порядка и без поддержки множественного перечисления (разделитель ;).
Проверь и исправь код, а при необходимости дополни лист language (object - это имя листа excel, при условии что type=sheet). Учитывай, что одно и тоже название поля на разных листах может иметь разный перевод (у них будут разные значения в поле object).
